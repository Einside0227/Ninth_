# 멀티플레이 숫자 야구 게임

언리얼 엔진의 멀티플레이 기능을 활용하여 제작한 턴제 숫자 야구 게임입니다.  
플레이어는 게임 시작 화면에서 이름을 입력한 뒤 게임을 시작할 수 있으며, 채팅 입력을 통해 3자리 숫자를 추측하며 정답을 맞히는 방식으로 진행됩니다.

## 프로젝트 소개

이 프로젝트는 언리얼 엔진의 네트워크 구조를 활용하여 **멀티플레이 숫자 야구 게임**을 구현한 과제입니다.  
단순한 숫자 판정 기능뿐 아니라, 서버와 클라이언트의 역할을 구분하고 플레이어별 상태와 공용 상태를 나누어 관리하는 구조를 직접 설계하는 데 중점을 두었습니다.

게임은 턴제 방식으로 진행되며, 플레이어는 자신의 턴에만 숫자를 입력할 수 있습니다.  
게임 시작, 턴 진행, 승리 판정, 재시작까지 모두 멀티플레이 환경에 맞게 구현하였습니다.

## 과제 목표

이번 과제의 목표는 언리얼 엔진에서의 멀티플레이 구조를 이해하고, 이를 기반으로 실제 게임 흐름을 설계하고 구현하는 것이었습니다.

주요 목표는 다음과 같습니다.

- 서버 권한 기반으로 게임이 진행되도록 구현
- 플레이어별 상태와 전체 게임 상태를 분리하여 관리
- 턴제 숫자 야구 규칙을 멀티플레이 환경에 맞게 적용
- UI와 게임 로직을 분리하여 구조적으로 설계
- 게임 시작 / 종료 / 재시작 흐름을 자연스럽게 연결

---

## 게임 규칙

- 정답은 **중복되지 않는 3자리 숫자**입니다.
- 숫자는 **1부터 9까지** 사용됩니다.
- 플레이어는 자신의 턴에 3자리 숫자를 입력합니다.
- 결과는 다음과 같이 판정됩니다.
  - **Strike**: 숫자와 위치가 모두 맞음
  - **Ball**: 숫자는 맞지만 위치가 다름
  - **OUT**: 맞는 숫자가 하나도 없음
- 정해진 시도 횟수 안에 정답을 맞히면 승리합니다.
- 모든 플레이어가 기회를 모두 사용하면 무승부가 됩니다.

## 주요 기능

### 1. 멀티플레이 턴제 진행
현재 턴인 플레이어만 숫자를 입력할 수 있도록 구현하였습니다.  
자신의 턴이 아닐 경우 입력이 제한되며 안내 메시지가 출력됩니다.

### 2. 채팅 입력 기반 숫자 판정
플레이어는 채팅 입력창에 3자리 숫자를 입력하여 게임을 진행합니다.  
입력값이 올바른 형식인지 검사한 뒤 스트라이크, 볼, 아웃을 판정합니다.

### 3. 플레이어별 상태 관리
플레이어 이름, 현재 시도 횟수, 최대 시도 횟수는 `PlayerState`를 통해 관리하였습니다.  
이를 통해 각 플레이어의 진행 상태를 독립적으로 처리할 수 있도록 하였습니다.

### 4. 시작/재시작 버튼을 누른 플레이어가 첫 턴
게임 시작 버튼을 누른 플레이어가 첫 번째 턴을 가지도록 구현하였습니다.
게임 종료 후 재시작 버튼을 누르면 게임이 초기화되며, 재시작을 누른 플레이어가 첫 번째 턴을 가지도록 구현하였습니다.

### 5. 결과 UI
게임 종료 시 승리 또는 무승부 결과를 결과 위젯을 통해 화면에 표시하도록 구현하였습니다.

### 6. 도중 접속 대응
게임이 이미 시작된 상태에서 접속한 플레이어는 시작 화면이 아닌 현재 게임 진행 상태를 이어서 볼 수 있도록 처리하였습니다.

## 구현 과정

### 1. PlayerController를 통한 입력 및 UI 관리
`PlayerController`에서는 로컬 플레이어의 UI를 생성하고, 입력 및 버튼 요청을 서버에 전달하는 역할을 담당하도록 구현하였습니다.

- 채팅 입력 위젯 생성
- 시작 위젯 생성
- 결과 위젯 생성
- 게임 시작 요청
- 재시작 요청
- UI 표시 / 숨김 처리

### 2. PlayerState를 통한 플레이어 개별 상태 관리
`PlayerState`에는 플레이어마다 독립적으로 관리되어야 하는 상태를 저장하였습니다.

- 플레이어 이름
- 현재 시도 횟수
- 최대 시도 횟수

이 구조를 통해 각 플레이어의 상태를 별도로 관리할 수 있었습니다.

### 3. GameMode를 통한 핵심 게임 로직 처리
숫자 생성, 턴 진행, 판정, 승리 처리와 같은 핵심 게임 로직은 `GameMode`에서 처리하였습니다.

- 정답 숫자 생성
- 현재 턴 플레이어 관리
- 입력값 검증
- 스트라이크 / 볼 / 아웃 판정
- 승리 / 무승부 판정
- 게임 재시작 처리

모든 핵심 로직을 서버에서 처리하도록 하여 멀티플레이 환경에서도 일관성 있는 결과가 나오도록 하였습니다.

### 4. GameState를 통한 공용 상태 동기화
현재 턴 플레이어 이름, 남은 시간, 게임 시작 여부와 같이 모든 플레이어가 함께 봐야 하는 정보는 `GameState`에서 관리하였습니다.  
이를 통해 새로 접속한 플레이어도 현재 게임 진행 상태를 확인할 수 있도록 구성하였습니다.

### 5. UI와 게임 로직 분리
입력과 표시를 담당하는 UI와 실제 게임 규칙을 처리하는 로직을 분리하여 설계하였습니다.  
이를 통해 코드의 역할이 명확해졌고, 수정이나 디버깅이 쉬워졌습니다.

## 핵심 구현 포인트

### 서버 권한 기반 처리
숫자 판정, 턴 진행, 승리 판정 등 게임의 핵심 로직은 서버에서만 처리하도록 구성하였습니다.  
이를 통해 클라이언트마다 다른 결과가 발생하지 않도록 하였습니다.

### 역할 분리
언리얼 멀티플레이 구조에 맞게 각 클래스의 역할을 나누어 구현하였습니다.

- `PlayerController` : 입력 및 UI 처리
- `PlayerState` : 플레이어 개별 상태 저장
- `GameMode` : 게임 규칙 및 진행 처리
- `GameState` : 공용 상태 동기화

### 입력 검증
숫자 입력 시 다음 조건을 검사하도록 구현하였습니다.

- 3자리 숫자인지
- 숫자만 포함되어 있는지
- 0이 포함되지 않았는지
- 중복 숫자가 없는지

## 트러블슈팅

### 1. 다른 클라이언트에서 UI가 함께 보이는 문제
초기에는 로컬 플레이어의 UI가 다른 클라이언트에도 영향을 주는 문제가 있었습니다.  
이를 해결하기 위해 `IsLocalController()`를 기준으로 로컬 플레이어에게만 UI를 생성하도록 수정하였습니다.

### 2. 시작 버튼을 누른 플레이어가 첫 턴이 되지 않던 문제
처음에는 기본 인덱스를 기준으로 턴이 시작되어, 버튼을 누른 플레이어가 첫 턴이 되지 않는 문제가 있었습니다.  
이를 해결하기 위해 버튼을 누른 `PlayerController`를 배열에서 찾아 해당 인덱스를 현재 턴으로 설정하도록 구현하였습니다.

### 3. 게임 중간에 접속한 플레이어에게 시작 화면이 보이던 문제
이미 게임이 시작된 상태에서도 새로 접속한 플레이어에게 시작 UI가 표시되는 문제가 있었습니다.  
이를 해결하기 위해 `GameState`의 게임 시작 여부를 기준으로 시작 위젯의 표시 여부를 제어하도록 수정하였습니다.
